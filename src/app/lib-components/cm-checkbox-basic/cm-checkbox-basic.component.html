Basic. #

To use this component, in your module you need to add: 
import { GlnCheckboxModule } from 'gelenium-ui';

The GlnCheckbox component displays the behavior of a simple "state checkbox" property (<input type="checkbox">). A component allows you to select one or more of these components.

For example:

Attributes. #

The following attributes are supported:

  • "id" - parameter for defining the unique identifier of the input field (id="unique_identifier");
  • "config" - Defines a configuration object. More details.
  • "isChecked" - specifies only the initial value of the element (isChecked);
  • "isDisabled" - a sign that the field is disabled (isDisabled);
  • "isReadOnly" - Specifies the status of an element as read-only (isReadOnly);
  • "isIndeterm" - Determines whether the checkbox is in the "indeterminate" state (isIndeterm);

Palette. #

The GlnCheckbox component has the ability to change its color.

For example: 
<gln-checkbox
  [glnColor]="{ primary: 'danger' }"
  formControlName="model02a">
  Label
</gln-checkbox>

This means that you want to use the system color "danger" as the "primary" color.

To use the glnColor directive, you need to add in your module: 
import { GlnColorModule } from 'gelenium-ui';

Config. #

If your application will have a single style for GlnCheckbox elements, then you can set a single configuration. To do this, you need to create an object with the GlnCheckboxConfig interface and define the required values. Next, specify the resulting object in the list of providers for the entire application (at the level of the AppComponent). 

import {
  GlnCheckboxConfig, GLN_CHECKBOX_CONFIG
} from 'gelenium-ui';

const glnCheckboxConfigDefault: GlnCheckboxConfig = {
  isNoRipple: true,
  position: 'start',
};

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.scss'],
  providers: [
    { provide: GLN_CHECKBOX_CONFIG, useValue: glnCheckboxConfigDefault }
  ],
})
export class AppComponent {
  title = 'demo';
}

After that, all input elements based on the "GlnCheckbox" component will have the same configuration.

2. Parameter values: "isNoRipple=false" and "position=end". #

Suppose we have a provider in our application with "GlnCheckboxConfig" and all input elements accept its default values. We can change these values for an individual element. To do this, you need to specify in this element those properties that do not match the provider's configuration.

For example, let's specify the following parameters: 

  <gln-checkbox
    isNoRipple="false"
    position="end"
    formControlName="model06b">Label
  </gln-checkbox>

In other words, there is the following order of determining the configuration parameters:

  • Standard values are taken (if there is one).
  • Values from the default configuration provider (if there is one).
  • Values from input parameters (if there is one).
