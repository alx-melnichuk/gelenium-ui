Basic. #

To use this component, in your module you need to add: 
import { GlnAutocompleteModule, GlnOptionModule } from 'gelenium-ui';

The GlnAutocomplete component allows you to supplement the input of the GlnInput component with a panel of suggested options.

For example: 
<gln-input formControlName="model01a"
  label="Enter value"
  [glnAutocompleteTrigger]="autocomplete01">
  </gln-input>
  <gln-autocomplete #autocomplete01="glnAutocomplete">
    <gln-option value="mango">Mango</gln-option>
    <gln-option value="lemon">Lemon</gln-option>
    <gln-option value="orange">Orange</gln-option>
    <gln-option value="kiwi">Kiwi</gln-option>
  <gln-autocomplete>

If you are using the GlnInput component, then you need to add the appropriate module. 

import { GlnInputModule } from 'gelenium-ui';

You can open the options panel by clicking the mouse, pressing the 'Arrow Up' or 'Arrow Down'.

You can close the options panel by clicking on another element, by pressing 'Escape' or 'Alt + Arrow Up'.

Asynchrony #

Synchronous processing: #

The GlnAutocomplete component displays a list of options on its panel. We can react to a change in the value in the input line and change the list of options. This will display, in the list of options, only the current values.

For a synchronous update, let's define a method for handling the "input" event in the input line. This method fires when the user enters new values. And in this method we will call the function of filtering the entire list of options by the value entered in the input line.

For example, this template: 
<gln-input formControlName="model03a"
  [glnAutocompleteTrigger]="autocomplete03a"
  (input)="value03a = filtered(fruits, control03c.model03a.value)">
</gln-input>
<gln-autocomplete #autocomplete03a="glnAutocomplete"
  [visibleSize]="5">
  <gln-option *ngFor="let item of value03a"
    [value]="item">
    {{ item }}
  </gln-option>
</gln-autocomplete>
For example, this component: 
export class AutocompleteComponent {

  public control03c = {
    model03a: new FormControl(null, []),
  };

  public formGroup03c: FormGroup = new FormGroup(this.control03c);
  public value03a: string[] = [];

  public filtered(list: string[] | null, value: string | null): string[] {
    const valueStr = (value || '').toLowerCase();
    return list?.filter((item) => item.toLowerCase().includes(valueStr)) || [];
  }
}

Asynchronous processing: #

For an asynchronous update, we do the same, only the filtering function will be asynchronous.

In asynchronous mode, the list of options is formed for some time. Only after the subscription to changes in the list of options has been triggered, you can open the options panel.

When performing an asynchronous load, the panel does not open and it is not clear to the user what is currently running.

We want to display the sign of data loading with the string: "Loading...". In order for the panel to open, let's add one option and along with it add the "<div>Loading...</div>" block.

Now, at the time of loading, a panel with the line "Loading ..." will be displayed. And so that the option that we added above is not visible, we indicate to it "display: none;".

For example, this template: 
<gln-input formControlName="model03c"
  [glnAutocompleteTrigger]="autocomplete03c"
  (input)="filteredAsyn(fruits, control03d.model03c.value, value03c$);">
  <span *ngIf="(value03c | async) == null"
    gln-orn-rg
    class="acs-spinner-sun-rays">
  </span>
</gln-input>
<gln-autocomplete #autocomplete03c="glnAutocomplete"
  [visibleSize]="6">
  <ng-container *ngIf="(value03c | async) == null">
    <gln-option style="display: none;">
    </gln-option>
    <div style="padding: 10px 10px;">
      Loading...
    </div>
  </ng-container>
  <gln-option *ngFor="let item of value03c | async"
    [value]="item">
    {{ item }}
  </gln-option>
</gln-autocomplete>
For example, this component: 
export class AutocompleteComponent {

  public control03d = {
    model03c: new FormControl(null, []),
    };

  public formGroup03d: FormGroup = new FormGroup(this.control03d);
  public value03c$: Subject<string[] | null>
    = new BehaviorSubject<string[] | null>([]);
  public value03c: Observable<string[] | null>
    = this.value03c$.asObservable();

  public filtered(list: string[] | null, value: string | null): string[] {
    const valueStr = (value || '').toLowerCase();
    return list?.filter((item) => item.toLowerCase().includes(valueStr)) || [];
  }
  public filteredAsyn(
    list: string[] | null,
    value: string | null,
    result$: Subject<string[] | null>
  ): void {
    const this2 = this;
    result$.next(null); // The null indicates that the data is being loaded.
    setTimeout(() => {
      result$.next(this2.filtered(list, value));
    }, 700);
  }
}

Feature #

The GlnAutocomplete component has the functionality of opening a panel to select available values. This panel closes when the user selects the desired value. Also, this panel is closed in case of loss of focus of the current input element. It is possible to forcibly close all open panels of GlnAutocomplete components, for example, when an emergency transition to another page occurs.

This situation is described in more detail in section Feature for GlnSelect.

Usage example:

1. set the scroll event handler for the parent component; # 

<div style="height: 100%; overflow-y: auto;"
  (scroll)="doScroll()">
  <div class="main-side-content">

    <gln-input
      [glnAutocompleteTrigger]="autocomplete04">
    </gln-input>
    <gln-autocomplete #autocomplete04="glnAutocomplete">
      <gln-option *ngFor="let item of value04" [value]="item">
        {{ item }}
      </gln-option>
    </gln-autocomplete>

    <gln-input
      [glnAutocompleteTrigger]="autocomplete05">
    </gln-input>
    <gln-autocomplete #autocomplete05="glnAutocomplete">
      <gln-option *ngFor="let item of value05" [value]="item">
        {{ item }}
      </gln-option>

    ...
  </div>

export class SiteComponent {
  public doScroll(): void {
    // Close the open GlnAutocomplete panel.
    GlnAutocompleteOpenUtil.closeAll();
  }
}

2. close the open panel of the GlnAutocomplete component;

There is a static class GlnAutocompleteOpenUtil, which contains a list of all GlnAutocomplete components with an open panel. Calling the static method "closeAll()" will gracefully close all open panes.

Highlight #

Sometimes it is required to select a fragment of the input line in the list of available values of the panel of the GlnAutocomplete component. You can use the "GlnHighlight" channel for this purpose.

It is required to add the "GlnHighlight" channel in the component template and pass the search value (input string value) as the first parameter to it.

Template example: 
<gln-input formControlName="model05"
  [glnAutocompleteTrigger]="autocomplete05">
</gln-input>
<gln-autocomplete #autocomplete05="glnAutocomplete">
  <gln-option *ngFor="let item of value05" [value]="item">
    <span
      [innerHtml]="item | glnHighlight: control05.model05.value">
    </span>
  </gln-option>
</gln-autocomplete>
Highlight of the input line.

Highlight for each word #

The "GlnHighlight" channel has a second optional parameter "isSearchByAllWords". If you specify this parameter, the search string will be split into separate fragments. And these fragments will be highlighted.

The division into fragments is performed by the following symbols: 

' ', '-', '[', ']', '/', '{', '}', '(', ')', '*', '+', '?', '.', '\', '^', '$', '|'
Template example: 
<gln-input formControlName="model05"
  [glnAutocompleteTrigger]="autocomplete05">
</gln-input>
<gln-autocomplete #autocomplete05="glnAutocomplete">
  <gln-option *ngFor="let item of value05" [value]="item">
    <span
      [innerHtml]="item | glnHighlight: control05.model05.value : true">
    </span>
  </gln-option>
</gln-autocomplete>
Highlight of the input line.

Config #

If your application will have a single style for GlnAutocomplete elements, then you can set a single configuration. To do this, you need to create an object with the GlnAutocompleteConfig interface and define the required values. Next, specify the resulting object in the list of providers for the entire application (at the level of the AppComponent). 

const glnAutocompleteConfigDefault: GlnAutocompleteConfig = {
  position: 'center',
  visibleSize: 8,
};

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.scss'],
  providers: [
    { provide: GLN_AUTOCOMPLETE_CONFIG, useValue: glnAutocompleteConfigDefault }
  ],
})
export class AppComponent {
  title = 'demo';
}

After that, all "GlnAutocomplete" components will have the same configuration.

2. Configuration with "position" and "visibleSize". #

Suppose we have a provider in our application with "GlnAutocompleteConfig" and all Autocomplete elements accept its default values. We can change these values for an individual element. To do this, you need to specify in this element those properties that do not match the provider's configuration.

For example, let's specify the following parameters: 

position="center"
visibleSize=5

In other words, there is the following order of determining the configuration parameters:

  • Standard values are taken. 
    isMaxWd=false,
isNoAnimation=false,
position='start',
visibleSize=0
  • Values from the default configuration provider (if there is one).
  • Values from input parameters (if there is one).
