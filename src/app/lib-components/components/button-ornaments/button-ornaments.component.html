Ornament for GrnButton in "button" element mode.

For the GrnButton component, it is possible to display an ornament (picture or text).

To display the ornament on the left, you need to add the "grn-orn-lf" attribute to the element inside the "grn-button" content.

Example: 
<grn-button>
  <div grn-orn-lf>
    <span>left</span>
  </div>
  Ornament on the left
</grn-button>

To display the ornament on the right, you need to add the "grn-orn-rg" attribute to the element inside the "grn-button" content.

Example: 
<grn-button>
  Ornament on the right
  <div grn-orn-rg>
    <span>right</span>
  </div>
</grn-button>

For blocks: "<span>left</span>" and "<span>right</span>" you need to specify the value of horizontal indents.

In this example, the css class "bo-text" defines the horizontal padding. The css class "bo-font-large" defines the font size as 1.4 em.

Let's consider an example when the ornament block displays: svg.

By analogy with the previous examples, you can display a picture in the ornament block. Since the current color changes depending on the exterior, the color of the picture should also change. We will use a picture in svg format. The html tag "use" will help us to insert the contents of the svg file into our template.

Here is an example: 
<span class="bo-svg">
  <svg>
    <use href="{file.svg#id}"></use>
  </svg>
</span>

This scheme replaces the svg-tag with the contents of the svg-file. And it turns out that our svg becomes part of the DOM document. And in this case, we can add a css style that will change the color for our svg fragment.

Here is our css style: 
.bo-svg>svg {
  fill: currentColor;
}

In this example, the "bo-svg" css class defines the horizontal padding and dimensions.

Ornament for GrnButton in "link" element mode.

Let's consider an example when the ornament block displays: text.

When GrnButton is in "link" mode, its content is determined only by the content of the "a" tag html. This means that the ornament block needs to be inserted into the ntml container of the "a" tag.

Here is our example: 
<i grn-orn-lf></i>
<a href="#">
  <span class="bo-text">&lt;</span>
  <span>Previous link (text)</span>
</a>

If the "<span class='bo-text'>&lt;</span>" fragment is located before the button name, then it will be displayed to the left of it.
The "<span>Previous link (text)</span>" fragment defines the name of the button.
The css class "bo-text" defines the horizontal padding.
The "<i grn-orn-lf></i>" fragment removes the left padding for the GrnButton.

And vice versa. 

<a href="#">
  <span>Next link (text)</span>
  <span class="bo-text">&gt;</span>
</a>
<i grn-orn-rg>>/i>

If the "<span class='bo-text'>&gt;</span>" fragment is located after the button name, then it will be displayed to the right of it.
The "<span>Next link (text)</span>" fragment defines the name of the button.
The css class "bo-text" defines the horizontal padding.
The "<i grn-orn-rg></i>" fragment removes the right padding for the GrnButton.
