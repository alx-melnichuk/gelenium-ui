Basic. #

To use this component, in your module you need to add: 
import { GlnRadioButtonModule, GlnRadioGroupModule } from 'gelenium-ui';

GlnRadioButton components are typically used in groups that represent possible values for a single parameter. Components are grouped by a unique "name" attribute value that is the same for all components in this group. In this case, only one value can be selected.

The GlnRadioButton component can be used in reactive forms.

For example: 
<form [formGroup]="formGroup01a" novalidate autocomplete="off">
  <gln-radio-button *ngFor="let item of fruits01a"
    formControlName="model01a"
    name="model01a"
    [value]="item"> {{ item }}
  </gln-radio-button>
</form>
With "formControlName" model="{{ formGroup01a.controls['model01a'].value }}""

Config. #

If your application will have a single style for GlnRadioButton elements, then you can set a single configuration. To do this, you need to create an object with the GlnRadioButtonConfig interface and define the required values. Next, specify the resulting object in the list of providers for the entire application (at the level of the AppComponent). 

import {
GlnRadioButtonConfig, GLN_RADIO_BUTTON_CONFIG
} from 'gelenium-ui';

const glnRadioButtonConfigDefault: GlnRadioButtonConfig = {
  isNoRipple: true,
  position: 'start',
};

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.scss'],
  providers: [
    { provide: GLN_RADIO_BUTTON_CONFIG, useValue: glnRadioButtonConfigDefault }
  ],
})
export class AppComponent {
  title = 'demo';
}

After that, all input elements based on the "GlnRadioButton" component will have the same configuration.

1. Configuration with "isNoRipple: true" and "position: start". #

For example, let's prepare a configuration in which:

  • "isNoRipple" will have the value "true";
  • "position" will have the value "start";).
const glnRadioButtonConfigDefault: GlnRadioButtonConfig = {
  isNoRipple: true,
  position: 'start',
};

2. Parameter values: "isNoRipple=false" and "position=end". #

Suppose we have a provider in our application with "GlnRadioButtonConfig" and all input elements accept its default values. We can change these values for an individual element. To do this, you need to specify in this element those properties that do not match the provider's configuration.

For example, let's specify the following parameters: 

  <gln-radio-button
    isNoRipple="false"
    position="end"
    formControlName="model06b">Label
  </gln-radio-button>

In other words, there is the following order of determining the configuration parameters:

  • Standard values are taken (if there is one).
  • Values from the default configuration provider (if there is one).
  • Values from input parameters (if there is one).
