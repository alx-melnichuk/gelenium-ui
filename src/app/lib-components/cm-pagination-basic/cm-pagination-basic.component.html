Basic. #

To use this component, in your module you need to add: 
import { GlnPaginationModule } from 'gelenium-ui';

The GlnPagination component is designed to change the current page from the specified range.

For example: 
<gln-pagination
  [page]="page01"
  [count]="pageCount01"
  (changePage)="page01a=$event">
</gln-pagination>
 The "page" current page number.
The "count" - total number of pages.
The "changePage" parameter is the page number change event.
Exterior: 'text'.
Exterior: 'outlined'.

Attributes. #

The following attributes are supported:

  • "id" - parameter for defining the unique identifier of the input field (id="unique_identifier");
  • "config" - Defines a configuration object. More details.
  • "count" - total number of pages; (default 1)
  • "page" - current page number; (default 1)
  • "exterior" - Determines the appearance of an element.
    It can take the following values:
    • "text" - default;
    • "outlined";
  • "isDisabled" - a sign that the field is disabled (isDisabled);
  • "isHidePrev" - indicates that the "previous page" button is hidden (isHidePrev);
  • "isHideNext" - indicates that the "next page" button is hidden (isHideNext);
  • "isShowFirst" - indicates that the "first page" button is displayed (isShowFirst);
  • "isShowLast" - indicates that the "last page" button is displayed (isShowLast);

Size #

The "size" property determines the size of the element (its height).

It takes the following values:

  • "short" - sets the height to 38px (default);
  • "small" - sets the height to 44px;
  • "middle" - sets the height to 50px;
  • "wide" - sets the height to 56px;
  • "large" - sets the height to 62px;
  • "huge" - sets the height to 68px;

The font size is determined from the parent container. The size of the font can be defined with the css-style for the "gln-pagination" tag. We can change the font size at the gln-pagination tag level, but the height of the element will not change.

The height of the element remains the same even when the font size is changed (from font-size: 10px to font-size: 26px).

It is possible to specify the height of an element. To do this, in the "config.size" property, we define the required height in pixels.

Border #

The GlnPagination component has the ability to change the border-radius css property.
In other words, the buttons of the GlnPagination component can be square instead of round.
The round shape of the GlnPagination component buttons is given a special css property '--glnbt-br-rd'.
If you exclude this css-property, then the shape of the buttons will be square.
To disable this css property, you need to add the 'isNoRounded' attribute.

Attributes: isNoRounded, Exterior: 'text'.
Config: isNoRounded, Exterior: 'outlined'.

Ornaments. #

For the GlnPagination component, it is possible to display an ornament (picture or text).
To display the ornament, you need to add a block inside "gln-pagination" with one of the attributes: "gln-orn-first", "gln-orn-prev", "gln-orn-next", "gln-orn-last".

Example: 
<gln-pagination>
  <span gln-orn-prev>prev</span>
  <span gln-orn-next>next</span>
  <span gln-orn-first>first</span>
  <span gln-orn-last>last</span>
</gln-pagination>

For these blocks, you need to specify a horizontal indent value or use the 'glnpg-pd-hor' attribute.

_#PC

Config. #

If your application will have a single style for all GlnPagination elements, then you can set a single configuration. To do this, you need to create an object with the GlnPaginationConfig interface and define the required values. Next, specify the resulting object in the list of providers for the entire application (at the level of the AppComponent). 

const glnPaginationConfigDefault: GlnPaginationConfig = {
  countNearby: 2,
  exterior: 'outlined',
  size: 'small',
};

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.scss'],
  encapsulation: ViewEncapsulation.None,
  providers: [
    { provide: GLN_PAGINATION_CONFIG, useValue: glnPaginationConfigDefault }
  ],
})
export class AppComponent {
  title = 'demo';
}

After that, all GlnPagination elements based on the "GlnPaginationConfig" component will have the same configuration.

GlnPaginationConfig interface. #

Name Description
count?: number; The total number of pages.
If a value less than zero is specified, then zero is used.
(default = 1;)
(optional)
countBorder?: number; The number of always visible pages at the beginning and at the end.
If a value less than zero is specified, then zero is used.
(default = 1;)

2. Specify "text" and "middle". #

Suppose we have a provider in our application with "GlnPaginationConfig" and all GlnPagination elements accept its default values. We can change these values for an individual element. To do this, you need to specify in this element those properties that do not match in the configuration provider.

For example, let's specify the following parameters: 

  count: 12,
  countNearby: 1,
  exterior="text",
  size="middle"

In other words, there is the following order of determining the configuration parameters:

  • Standard values are taken.

  exterior="text",
  isHideNext=false,
  isHidePrev=false,
  isNoRounded=false,
  isShowFirst=false,
  isShowLast=false,
  size="short"
  • Values from the default configuration provider (if there is one).
  • Values from input parameters (if there is one).
