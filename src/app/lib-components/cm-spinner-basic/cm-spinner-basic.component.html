Basic. #

To use this component, in your module you need to add: 
import { GlnSpinnerModule } from 'gelenium-ui';

The GlnSpinner component is used to display the progress of data processing.
This component renders an internal "svg" image that rotates around its own axis.
Also, this component can display its content instead of an internal image.

Attributes. #

The following attributes are supported:

  • "id" - parameter for defining the unique identifier of the input field (id="unique_identifier");
  • "config" - Defines a configuration object. More details.
  • "isExternal" - Displays the specified content instead of an inline SVG image.
  • "isNoAnimation" - disable animation execution (isNoAnimation);
  • "isNoPulsate" - disable circle pulsate (isNoPulsate);
  • "size" - specifies the size (height and width) of the component.

  • "wide" - sets the height to 56px;
  • "large" - sets the height to 62px;

Customization. #

The GlnSpinner component has additional user customization. By default, the GlnSpinner component renders an inline svg image. It is also possible to display user-specified content. The 'isExternal' parameter is used for this.

You can specify in the content a class that uses the svg image from resources.

For example html: 
<gln-spinner isExternal>
  <i class="app-external-svg"></i>
</gln-spinner>
 For example css: 
.app-external-svg {
  background-image: url('assets/icons/icon-demo.svg');
  display: inline-block;
  height: inherit;
  width: inherit;
}

You can also specify in the content the 'svg' tag inside which the 'use' tag. And already inside the 'use' tag, use a link to the svg image from the resources.

For example html: 
<gln-spinner isExternal>
  <svg><use href="assets/icons/icon-demo.svg"></use></svg>
</gln-spinner>

Config #

If your application will have a single style for GlnSpinner elements, then you can set a single configuration. To do this, you need to create an object with the GlnSpinnerConfig interface and define the required values. Next, specify the resulting object in the list of providers for the entire application (at the level of the AppComponent). 

const glnSpinnerConfigDefault: GlnSpinnerConfig = {
  exterior: 'underline',
  size: 'wide',
};

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.scss'],
  providers: [
    { provide: GLN_SPINNER_CONFIG, useValue: glnSpinnerConfigDefault }
  ],
})
export class AppComponent {
  title = 'demo';
}

After that, all "GlnSpinner" components will have the same configuration.

2. Configuration with "middle". #

Suppose we have a provider in our application with "GlnSpinnerConfig" and all "GlnSpinner" elements accept its default values. We can change these values for an individual element. To do this, you need to specify in this element those properties that do not match the provider's configuration.

For example, let's specify the following parameters: 

size="middle"

In other words, there is the following order of determining the configuration parameters:

  • Standard values are taken. 
    isExternal=false,
isNoAnimation=false,
isNoPulsate=false,
size='small'
  • Values from the default configuration provider (if there is one).
  • Values from input parameters (if there is one).
1. Default config: "short".
2. size="middle".
