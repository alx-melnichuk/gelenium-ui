Basic. #

To use this component, in your module you need to add: 
import { GlnTooltipModule } from 'gelenium-ui';

The GlnTooltip component displays a tooltip when the user hovers over or long presses on an element.

For example: 
<button glnTooltip="Information">
  Action
</button>
Show tooltip on hover.

  public incVal(value: number, delta: number, max: number): number {
    return value + delta <= max

_Feature_

Feature #

Sometimes there are situations where we don't know if an element will fit entirely within its parent's bounds. This may be when internationalization is used, i.e. different translations of the value are substituted. Or when the value does not fit into the specified table column width. In these cases, we should only display the tooltip when the element goes beyond its parent.
To do this, you can use the glnOverflowChild directive.
The directive's getChildHasOverflow() method determines the first child element and checks if it overflows its parent. And if it goes beyond the boundaries of the parent element, then we can enable the display of a tooltip.

For example: 
<button class="ttbs-btn"
  style="max-width: 75px; white-space: nowrap; overflow: hidden;"
  glnTooltip="Action demo"
  [glnttDisabled]="!overflowChild1.getChildHasOverflow()"
  glnOverflowChild
  #overflowChild1="glnOverflowChild">
  <span>Action demo</span>
</button>

_Config_

Config #

If your application will have a single style for GlnTooltip elements, then you can set a single configuration. To do this, you need to create an object with the GlnTooltipConfig interface and define the required values. Next, specify the resulting object in the list of providers for the entire application (at the level of the AppComponent). 

const glnTooltipConfigDefault: GlnTooltipConfig = {
  classes: 'ttc-demo',
  isArrow: true,
};

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.scss'],
  providers: [
    { provide: GLN_TOOLTIP_CONFIG, useValue: glnTooltipConfigDefault }
  ],
})
export class AppComponent {
  title = 'demo';
}

After that, all "GlnTooltip" components will have the same configuration.

Providers. #

For "GlnTooltip" it is possible to change the scroll behavior. To do this, you need to define your provider with the name "GLN_TOOLTIP_SCROLL_STRATEGY".

Here is an example of usage 
export function GLN_TOOLTIP_SCROLL_STRATEGY_PROVIDER_BLOCK_FACTORY(
  overlay: Overlay
): () => ScrollStrategy {
  return () => overlay.scrollStrategies.block();
}

export const GLN_TOOLTIP_SCROLL_STRATEGY_PROVIDER_BLOCK = {
  provide: GLN_TOOLTIP_SCROLL_STRATEGY,
  deps: [Overlay],
  useFactory: GLN_TOOLTIP_SCROLL_STRATEGY_PROVIDER_BLOCK_FACTORY,
};

@NgModule({
  declarations: [AppComponent],
  imports: [BrowserModule, BrowserAnimationsModule, ...],
  bootstrap: [AppComponent],
  providers: [GLN_TOOLTIP_SCROLL_STRATEGY_PROVIDER_BLOCK],
})
export class AppModule {}

After that, all "GlnTooltip" components will have the same scroll behavior.
