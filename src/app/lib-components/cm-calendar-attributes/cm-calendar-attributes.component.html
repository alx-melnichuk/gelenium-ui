Attributes. #

The following attributes are supported:

  • "isDisabled" - a sign that the field is disabled (isDisabled);
  • "isHideOldDays" - a sign of hiding the days of the previous and next month (isHideOldDays);
  • "isHorizont" - renders a component with a horizontal layout (isHorizont);
  • "isReadOnly" - a sign that the field is read-only (isReadOnly);
Attribute: isDisabled.
Attribute: isHorizont.
Attribute: isReadOnly.
Attribute: isHideOldDays.
source

Attributes are also supported 2: #

  • "wdFull" - an attribute sets the width to the width of the parent (width: 100%).
    In this case, the width must be defined at the parent or at the current level.
    Without this parameter, the width is determined by the content and may change when the font size is changed.
    To make the child element have the height of the parent, we specify "display: grid;" for a parent.
  • "sizeDayWeek" - specifies the length of the name of the day of the week ([sizeDayWeek]="2")
Attribute: wdFull (min-width: 320px; min-height: 320px;).
Attribute: wdFull, isHorizont (min-width: 320px; min-height: 320px;).
Attribute: sizeDayWeek=1 ('M').
Attribute: sizeDayWeek=3 ('Mon').

Attributes are also supported 3: #

  • "isHideDayoff" - hide display of dayoffs in "danger" color (isHideDayoff);
  • "isStartSunday" - specifies the start of the week from "Sunday".
    By default, the start of the week is determined by the locale.
  • "isWeekNumber" - specifies the display of the week number (isWeekNumber);
Attribute: isHideDayoff.
Attribute: isWeekNumber.

Attributes are also supported 4: #

  • "isTwoDigitDay" - defines a two-digit day value (isTwoDigitDay);
  • "formatMonth" - defines the format of the month name in the header.
    takes the following values:
    • "numeric" - "6";
    • "2-digit" - "06";
    • "long" - "June";
    • "short" - "Jun" (default);
    • "narrow" - "J";
  • "views" - defines an array of available values for the display mode.
    (default = ['day', 'year', 'month'];)
    takes the following values:
    • 'day' - display mode for day selection;
    • 'month' - display mode for month selection;
    • 'year' - display mode for year selection;
  • "view" - defines the initial value of the display mode.
    takes the following values:
    • 'day' - display mode for day selection;
    • 'month' - display mode for month selection;
    • 'year' - display mode for year selection;
Attribute: isTwoDigitDay.
Attribute: formatMonth='2-digit'.
Attribute: views=['month', 'year'], view='year'.
Attribute: views=['month', 'year'].
source

Attributes are also supported 5: #

  • "formatByMonths" - defines the format for naming months in a grid of months (for view='month');
    takes the following values:
    • "numeric" - "6";
    • "2-digit" - "06";
    • "long" - "June";
    • "short" - "Jun" (default);
    • "narrow" - "J";
  • "colsByMonths" - defines the number of columns in the months grid.
    [1..12] (for view='month'). (default = 3;)
  • "colsByYears" - defines the number of columns in the years grid.
    [1..12] (for view='year'). (default = 4;)
  • "rowsByYears" - defines the number of rows in the years grid.
    [1..12] (for view='year'). (default = 5;)

Attributes are also supported 6: #

  • "dateClasses" - function that can be used to add custom CSS classes to dates; 
    dateClasses1 = (date: Date, view: string, currentDate: Date): string[] => {
  const result: string[] = [];
  const day: number = date.getDate();
  if ('day' === view && date.getMonth() === currentDate.getMonth()
    && (day === 1 || day === 20)) {
    result.push('cnat6-demo1');
  }
  return result;
};
    (default = undefined;)
  • "dateDisabled" - function that can be used to disable a date based on a custom condition; 
    dateDisabled1 = (date: Date, view: string, currentDate: Date): boolean => {
  const dayWeek: number = date.getDay();
  return 'day' ==== view && date.getMonth() === currentDate.getMonth()
    && (dayWeek === 0 || dayWeek === 1);
};
    (default = undefined;)
