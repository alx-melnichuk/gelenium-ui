Feature #

Suppose we have a parent component that consists of several sections (child components). Each section has its own unique id (#anchor) and is at least 800px high. An absolute link to an anchor allows you to quickly navigate to the desired section. In other words, using the link "href=#anchor" we can move between sections.

Let's assume that the user followed an absolute link to the first section. The user then followed an absolute link to the second section. In the second section, the user opened the GlnSelect component panel.

Next, the user presses a button on the "Back" panel (go to the previous link, i.e. return to the first section). The panel of the GlnSelect component from the second section will open on the screen.

This happens because the component of the second section was not destroyed and there was no event to close the open panel of the GlnSelect component. In this case, the parent component was not explicitly scrolled. Although all GlnSelect components were set to "block scroll" when the panel was open, it didn't work in this case. And since the GlnSelect component's panel is rendered on its own layer of the "body" element, this results in an overlay of the first section and the open GlnSelect component's panel of the second section. (This issue arose while developing this demo page.)

To solve this problem, you need:

1. set the scroll event handler for the parent component; 
<div style="height: 100%; overflow-y: auto;"
  (scroll)="doScroll()">
  <div class="main-side-content">
    <section id="section1">
      <app-section1></app-section1>
    </section>
    <section id="section2">
      <app-section2>
        ...
        <gln-select></gln-select>
        ...
      </app-section2>
    </section>
    ...
  </div>
</div>
 
export class SiteComponent {
  public doScroll(): void {
    // Close an open panel.
    GlnSelectOpenUtil.closeAll();
  }
}

2. close the open panel of the GlnSelect component;

There is a static class GlnSelectOpenUtil, which contains a list of all GlnSelect components with an open panel. Calling the static method "closeAll()" will gracefully close all open panes.
