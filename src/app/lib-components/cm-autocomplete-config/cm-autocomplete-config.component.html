Config #

If your application will have a single style for GlnAutocomplete elements, then you can set a single configuration. To do this, you need to create an object with the GlnAutocompleteConfig interface and define the required values. Next, specify the resulting object in the list of providers for the entire application (at the level of the AppComponent). 

const glnAutocompleteConfigDefault: GlnAutocompleteConfig = {
  position: 'center',
  visibleSize: 7,
};

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.scss'],
  providers: [
    { provide: GLN_AUTOCOMPLETE_CONFIG, useValue: glnAutocompleteConfigDefault }
  ],
})
export class AppComponent {
  title = 'demo';
}

After that, all "GlnAutocomplete" components will have the same configuration.

1. Configuration with "classes", "position" and "visibleSize". #

For example, let's prepare a configuration in which:

  • "classes" will have the value ['acc-panel']
  • "position" will have the value 'end'
  • "visibleSize" will have the value 7
const glnAutocompleteConfigDefault: GlnAutocompleteConfig = {
classes=['acc-panel']
  position: 'end';
  visibleSize: 7
};

2. Configuration with "position" and "visibleSize". #

Suppose we have a provider in our application with "GlnAutocompleteConfig" and all Autocomplete elements accept its default values. We can change these values for an individual element. To do this, you need to specify in this element those properties that do not match the provider's configuration.

For example, let's specify the following parameters: 

position="center"
visibleSize=5

In other words, there is the following order of determining the configuration parameters:

  • Standard values are taken. 
    isClearOnEscape=false,
isMaxWd=false,
isNoAnimation=false,
isNoCloseOnSelect=false,
isNoOpenOnMouse=false,
isNoRipple=false,
isOpenOnFocus=false,
classes='',
position='start',
visibleSize=0
  • Values from the default configuration provider (if there is one).
  • Values from input parameters (if there is one).

import {
  Component, ViewEncapsulation
} from '@angular/core';
import {
  FormGroup, FormControl 
} from '@angular/forms';

@Component({
  selector: 'app-autocomplete',
  templateUrl: './autocomplete.component.html',
  styleUrls: ['./autocomplete.component.scss'],
  encapsulation: ViewEncapsulation.None,
})
export class AutocompleteComponent {
  // prettier-ignore
  public fruits = [
    'apple'     , 'apricot'  , 'avocado'   , 'banana'    , 'black currant',
    'blackberry', 'blueberry', 'cherry'    , 'coconut'   , 'cranberry'    ,
    'dragonfruit', 'fig'     , 'goji berry', 'grape'     , 'grapefruit'   ,
    'grapes'    , 'guava'    , 'iwi'       , 'jackfruit' , 'jamun'        ,
    'jujube'    , 'lemon'    , 'lime'      , 'longan'    , 'loquat'       ,
    'lychee'    , 'mandarin' , 'mango'     , 'melon'     , 'mulberry'     ,
    'nectarine' , 'kiwi'     , 'olive'     , 'orange'    , 'palm fruit'   ,
    'papaya', 'passion fruit', 'peach'     , 'pear'      , 'persimmon'    ,
    'pineapple' , 'plum'  , 'pomegranate', 'prickly pear', 'pumpkin'      ,
    'quince'    , 'raspberry', 'red currant', 'sapodilla', 'satsuma'      ,
    'strawberry','sweet lemon', 'tamarind' , 'tangerine' , 'watermelon'
  ];

  public control08a = {
    model08a: new FormControl(null, []),
    model08b: new FormControl(null, []),
  };
  public formGroup08a: FormGroup = new FormGroup(this.control08a);

  public capitalizeFirstLetter(value: string): string {
    return value ? value[0].toUpperCase() + value.slice(1) : '';
  }
}
