Border radius.

You can change the value of the rounding radius in outlined appearance mode. To do this, you need to specify a new value for the rounding radius in the "--gfi-o-br-rd" css-variable for the "grn-frame-input" css-style. For example: 

  .fibr-outlined-radius grn-frame-input {
    --gfi-o-br-rd: calc(var(--gfi-size)*0.5);
  }

This value for the fillet radius corresponds to 50% of the height of the current element. The minimum value for the fillet radius is 0. The maximum value for the fillet radius is 50% of the height of the current element. The value of the css-variable "--gfi-size" determines the height of the current element.

In this case, you also need to change the value of the left indent for correct display. The value of the left indentation is determined by the special css-variable "?--gfi-o-lbl-pd" in the css-style "grn-frame-input". For example: 

  .fibr-outlined-radius grn-frame-input {
    --gfi-o-br-rd: calc(var(--gfi-size)*0.5);
  }

In our example, the value of the coefficient for changing the fillet radius will change dynamically. For this, an additional css-variable "--fibr-border-radius" is used. As a result, we get the following: 

  .fibr-outlined-radius grn-frame-input {
    --gfi-o-br-rd: calc(var(--gfi-size)*var(--fibr-border-radius)/100);
  }

This is what the original values look like: 

  grn-frame-input {
    --gfi-o-br-rd: calc(var(--gfi-size)/10);
  }

Similarly, you can change the fillet radius for the underline appearance mode.

To do this, you need to specify a new value for the rounding radius in the css-variable "?--gfi-u-br-rd" for the css-style "grn-frame-input". And you also need to change the value of the left indentation value of the css-variable "?--gfi-u-lbl-pd" for the css-style "grn-frame-input".

We get the following: 

    .fibr-underline-radius grn-frame-input {
      --gfi-u-br-rd: calc(var(--gfi-size)*var(--fibr-border-radius)/100);
    }

This is what the original values look like: 

    grn-frame-input {
    --gfi-u-br-rd: calc(var(--gfi-size)/10);
    }

Border radius in "outlined" mode.

For the "outlined" mode, the value of the border rounding radius is set by the css variable "--gfi-o-br-rd".
By default, this variable has the following meaning: 

--gfi-o-br-rd: calc(var(--gfi-size)/10);

This variable is further used as follows: 

border-radius: var(--gfi-o-br-rd);

Thus, we can assign another valid value to this variable according to the description in " border-radius". It is also necessary to change the value of the left padding so that the label does not intersect with the border. For "outline" mode, the left and right padding values are set by the css variable "?--gfi-o-lbl-pd".
By default, this variable has the following meaning: 

?--gfi-o-lbl-pd: calc(0.25*var(--gfi-size));

Border radius in "underline" mode.

For the "underline" mode, the value of the border rounding radius is set by the css variable "--gfi-u-br-rd".
By default, this variable has the following meaning: 

--gfi-u-br-rd: calc(var(--gfi-size)/10) calc(var(--gfi-size)/10) 0 0;

This variable is further used as follows: 

border-radius: var(--gfi-u-br-rd);

Thus, we can assign another valid value to this variable according to the description in " border-radius".

It is also necessary to change the value of the left padding so that the label does not intersect with the border. For "underline" mode, the left and right padding values are set by the css variable "?--gfi-u-lbl-pd".
By default, this variable has the following meaning: 

?--gfi-u-lbl-pd: calc(0.214285714*var(--gfi-size));

If in the "underline" mode we want to round the bottom corners, then we additionally need to specify the height of the block with the bottom frame.

Here's an example: 

.fibr-radius-h grn-frame-input {
  --gfi-u-br-rd: 0 0 0 calc(var(--gfi-size)/3);
  .gfi-u-frm:before, .gfi-u-frm:after {
    border-radius: var(--gfi-u-br-rd);
    height: calc(0.65*var(--gfi-size));
  }
}
