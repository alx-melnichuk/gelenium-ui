Border radius.

You can change the value of the rounding radius in outlined appearance mode. This requires specifying a value for the "border-radius" property for the "grn-frame-input".
For example: 

  .fibr-outlined-radius .gfi-outlined {
    border-radius: calc(var(--size)*0.5);
  }

This value for the fillet radius corresponds to 50% of the height of the current element. The minimum value for the fillet radius is 0.
The maximum value for the fillet radius is 50% of the height of the current element.
The value of the css-variable "--size" determines the height of the current element.

In this case, you also need to change the value of the left indent for correct display. The value of the left indent is determined by the configuration for each size, respectively.
For example: 

  public configOutlinedShort: GrnFrameInputConfig = {
    labelPd: 21.66, // 0.57*FrameSizeValue.short
  };
  public configOutlinedSmall: GrnFrameInputConfig = {
    labelPd: 25.08, // 0.57*FrameSizeValue.small
  };
  ...
  public configOutlinedHuge: GrnFrameInputConfig = {
    labelPd: 38.76, // 0.57*FrameSizeValue.huge
  };

  public configUnderlineShort: GrnFrameInputConfig = {
    labelPd: 15.2, // 0.4*FrameSizeValue.short
  };
  public configUnderlineSmall: GrnFrameInputConfig = {
    labelPd: 17.6, // 0.4*FrameSizeValue.small
  };
  ...
  public configUnderlineHuge: GrnFrameInputConfig = {
    labelPd: 27.2, // 0.4*FrameSizeValue.huge
  };

In our example, the value of the coefficient for changing the fillet radius will change dynamically. For this, an additional css-variable "--fibr-border-radius" is used. As a result, we get the following: 

  .fibr-outlined-radius .gfi-outlined {
    border-radius: calc(var(--size)*var(--fibr-border-radius)/100);
  }

Initial value of the fillet radius: 

border-radius: calc(var(--size)/10);

Similarly, you can change the fillet radius for the underline appearance mode.

To do this, you also need to specify the value of the "border-radius" property for the "grn-frame-input" css style. And you also need to change the value of the left indentation. How we did it above by specifying the required indentation in the configuration.

We get the following: 

  .fibr-underline-radius .gfi-underline {
    --fibr-u-br-rd: calc(var(--size)*var(--fibr-border-radius)/100);
    border-radius: var(--fibr-u-br-rd) var(--fibr-u-br-rd) 0 0;
  }

Border radius in "outlined" mode.

Thus, to describe the radius of the rounding, we can assign any valid (in accordance with the description of "border-radius") value.
It is also necessary to change the value of the left padding so that the label does not intersect with the border.

Border radius in "underline" mode.

For the "underline" mode, the value of the border rounding radius is set in the same way.

Initial value of the fillet radius: 

border-radius: calc(var(--size)/10) calc(var(--size)/10) 0 0;

And in this case, we can use any valid value as described in "border-radius".
It is also necessary to change the value of the left padding so that the label does not intersect with the border.

If in the "underline" mode we want to round the bottom corners, then we additionally need to specify the height of the block with the bottom frame.

Here's an example: 

  .fibr-radius-h .gfi-underline {
    border-radius: 0 0 0 calc(var(--size)/3);
  }
  .fibr-radius-h .gfi-underline .gfi-frame:before,
  .fibr-radius-h .gfi-underline .gfi-frame:after {
    height: calc(0.65*var(--size));
  }
