Border radius.

You can change the value of the rounding radius in outlined appearance mode. To do this, you need to specify a new value for the rounding radius in the "‑‑gfi‑outlined-border-radius" css-variable for the "grn‑frame-input" css-style. For example: 

  .ibr-outlined-radius grn-frame-input {
    --gfi-outlined-border-radius: calc(var(--gfi-base-size)*0.5);
  }

This value for the fillet radius corresponds to 50% of the height of the current element. The minimum value for the fillet radius is 0. The maximum value for the fillet radius is 50% of the height of the current element. The value of the css-variable "‑‑gfi‑base-size" determines the height of the current element.

In this case, you also need to change the value of the left indent for correct display. The value of the left indentation is determined by the special css-variable "‑‑gfi-outlined-frm-pd-lf-rg" in the css-style "grn-frame-input". For example: 

  .ibr-outlined-radius grn-frame-input {
    --gfi-outlined-border-radius: calc(var(--gfi-base-size)*0.5);
    --gfi-outlined-frm-pd-lf-rg: calc(var(--gfi-base-size)*0.57);
  }

In our example, the value of the coefficient for changing the fillet radius will change dynamically. For this, an additional css-variable "‑‑ibr-border-radius" is used. As a result, we get the following: 

  .ibr-outlined-radius grn-frame-input {
    --gfi-outlined-border-radius:
      calc(var(--gfi-base-size)*var(--ibr-border-radius)/100);
    --gfi-outlined-frm-pd-lf-rg: calc(0.57*var(--gfi-base-size));
  }

This is what the original values look like: 

  grn-frame-input {
    --gfi-outlined-border-radius: calc(var(--gfi-base-size)/10);
    --gfi-outlined-frm-pd-lf-rg: calc(0.25*var(--gfi-base-size));
  }

Similarly, you can change the fillet radius for the underline appearance mode.

To do this, you need to specify a new value for the rounding radius in the css-variable "‑‑gfi-underline-border-radius" for the css-style "grn‑frame-input". And you also need to change the value of the left indentation value of the css-variable "‑‑gfi-underline-frm-pd-lf-rg" for the css-style "grn‑frame-input".

We get the following: 

    .ibr-underline-radius grn-frame-input {
      --gfi-underline-border-radius:
        calc(var(--gfi-base-size)*var(--ibr-border-radius)/100);
      --gfi-underline-frm-pd-lf-rg: calc(0.4*var(--gfi-base-size));
    }

This is what the original values look like: 

    grn-frame-input {
    --gfi-underline-border-radius: calc(var(--gfi-base-size)/10);
    --gfi-underline-frm-pd-lf-rg: calc(0.214285714*var(--gfi-base-size));
    }
