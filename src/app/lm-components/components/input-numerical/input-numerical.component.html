Numerical value

When using an input element with type="number", there are the following problems:

  • non-numeric values are allowed ('e', '+', '-', '.');
  • it is not very convenient to use scrolling to increase / decrease the value;

As a solution to this problem, it is suggested to use type="text" and pattern="^[0-9]+$". This allows the error status to be displayed when entering non-numeric values.

An alternative is to use the grnOnlyByRegex directive. This directive allows you to enter only those values that match the specified regular expression. The value of the onlyByRegex parameter is passed to the specified directive that is added to the input element. To enter only numbers, you can use: onlyByRegex="^[\d]+$" or [onlyByRegex]="'^[\\d]+$'".

The following templates can be specified in the grnOnlyByRegex directive as an input parameter:

  • '#numeric' - which matches the regular expression
    '^-?(\d+)$';
  • '#numeric(,2)' - which matches the regular expression
    '^-?(\d+(\.\d{0,2})?|\.\d{0,2})$';
  • '#numeric(12,2)' - which matches the regular expression
    '^-?(\d{1,12}(\.\d{0,2})?|\.\d{0,2})$';

You can also specify your own regular expressions as an input parameter. For example: numbers, lowercase characters, and space. [onlyByRegex]="'^[\ ,\d,a-z]+$'"
